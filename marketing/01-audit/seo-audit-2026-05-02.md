# Audit SEO — 02 mai 2026

> Audit technique + on-page d'Immatcalc 7 jours après le sprint Growth.
> Objectif : valider que les fondations sont saines + identifier les vrais axes
> d'amélioration restants.

## ✅ Ce qui est bon

### Technique

- **Compression gzip** active (49 KB → 9.7 KB sur la home, 80 % de réduction)
- **CDN cache** : `x-nextjs-cache: HIT` + `cache-control: s-maxage=31536000` sur les assets
- **Security headers** complets : HSTS (max-age=2y, preload), X-Frame, X-Content-Type, Referrer-Policy, Permissions-Policy
- **Cache assets** : `immutable` 1 an sur CSS/JS (parfait pour Core Web Vitals)
- **HTML léger** : 49 KB sur la home, 50-60 KB sur les pages internes — bien sous le seuil de 200 KB

### Indexation

- **Sitemap** : 42 URLs déclarées, 10/10 retournent 200 sur sample aléatoire
- **Robots.txt** : `User-Agent: *` Allow + bloque API/admin + bloque GPTBot/ClaudeBot/CCBot/anthropic-ai/Google-Extended (anti-LLM-training)
- **Canonical URLs** présentes sur toutes les pages internes testées (7/8)
- **HTTP 200** sur 9/9 pages clés testées

### On-page

- **Titles ≤ 60 chars** sur toutes les pages testées (sprint Day 2 confirmé)
- **JSON-LD** : 3 blocs sur la home (Organization, WebApplication, FAQPage)
- **H1 unique** par page (1 par page testée, jamais 0 ni 2)
- **Pas d'images** = pas d'alt manquants à corriger

## ❌ Ce qu'il faut corriger

### Priorité 1 — Thin content (10 pages 400-500 mots)

Le sprint Day 5 a étendu **3 pages** à 1200+ mots (Tesla Model Y, Renault Clio,
Île-de-France) mais a oublié les **autres pages modèles + 8 régions**.

| Page                                    | Mots actuels | Cible |
| --------------------------------------- | ------------ | ----- |
| /carte-grise/peugeot-208                | 471          | 800+  |
| /carte-grise/dacia-sandero              | 453          | 800+  |
| /carte-grise/citroen-c3                 | 441          | 800+  |
| /carte-grise/renault-captur             | 437          | 800+  |
| /carte-grise/auvergne-rhone-alpes       | 421          | 1000+ |
| /carte-grise/occitanie                  | 409          | 1000+ |
| /carte-grise/bretagne                   | 411          | 1000+ |
| /carte-grise/grand-est                  | ~410         | 1000+ |
| /carte-grise/hauts-de-france            | ~410         | 1000+ |
| /carte-grise/nouvelle-aquitaine         | ~410         | 1000+ |
| /carte-grise/pays-de-la-loire           | ~410         | 1000+ |
| /carte-grise/provence-alpes-cote-d-azur | ~410         | 1000+ |

**Pourquoi c'est le P1** : Google considère < 600 mots comme thin content sur des sujets
transactionnels/financiers (carte grise = "Your Money Your Life"). Les pages
Tesla Model Y + Clio + IDF risquent de **diluer** la moyenne du site si les autres
restent à 400 mots.

**Effort** : 4-5 heures pour étendre les 4 modèles + 8 régions, en réutilisant le
pattern `extraContent` slot déjà existant dans `ModelPage` et `RegionPage`.

### Priorité 2 — Canonical manquant sur la home

Toutes les pages internes ont `<link rel="canonical">` sauf **la home** elle-même.
Pas catastrophique (Google détecte naturellement la home) mais à corriger pour
cohérence et éviter les duplicates URL parameters (`?utm_*` etc.).

**Effort** : 5 min — ajouter `alternates: { canonical: '/' }` dans `apps/web/app/layout.tsx`.

### Priorité 3 — OG image manquante

Aucune des 7 pages testées n'a `og:image`. Ça impacte :

- Le partage LinkedIn / Twitter / Slack (pas de prévisu visuelle)
- Légèrement le ranking (signal social qualité)
- L'audit SEO de chiffre (Lighthouse SEO va flagger)

**Effort** : 30 min — créer l'asset 1200×630 (cf. `marketing/03-production/social/visuels-specs.md`
section "Card OpenGraph V1") et l'ajouter dans `metadata.openGraph.images` du layout.

### Priorité 4 — Pas de breadcrumb visible

Les breadcrumbs JSON-LD existent (vu dans le code SEO) mais aucune navigation visible
dans les pages. Google affiche les breadcrumbs dans les SERPs quand ils sont visibles
côté UI, ce qui améliore le CTR.

**Effort** : 1 heure — composant `<Breadcrumbs>` réutilisable + injection dans
`ArticleShell` qui prend déjà la donnée en input.
